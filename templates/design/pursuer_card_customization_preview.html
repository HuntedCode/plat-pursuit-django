Pursuer Card · Customization Design Workshop

Variants for each customization slot.

Sibling deep-dive to the main Pursuer Card workshop. The parent workshop named the five customization slots in Section 6; this one workshops what actually fills them. Every variant below is a concrete option the team can pick to ship. Badge-peek customization is deferred — covers four of the five slots for now.

The locked rule from the parent workshop applies throughout: customization is ADDITIVE on top of the tier-earned base, never replacing it. A Silver Pursuer cannot equip Platinum chrome — the tier is achievement-locked. Cosmetics add personality within the tier the player earned.

How a specific variant gets unlocked (gameplay milestone, customization currency, subscription perk, seasonal event) is intentionally out of scope here. That's an economy decision for later. This workshop just decides which variants are worth shipping as concrete looks.

All sample cards below render at Gold tier with Master Level 847, so the variants read against a consistent base. In production, every variant ships against all four tiers — the same "frame overlay glow" applies its glow on Bronze, Silver, Gold, and Platinum chrome, just tier-tinted.

01 Overview

How the slots work.

Each slot section below shows 3–5 variants of one cosmetic surface. The first card in each grid is always the default appearance that ships to every Pursuer at signup. The remaining cards are the alternative looks the player can equip on top.

How any specific variant is unlocked (gameplay milestone, customization currency, subscription perk, seasonal event) is intentionally not a property of this workshop. That's an economy decision that comes later. The purpose here is to lock the visual menu.

Variants from different slots stack. A Pursuer can run a textured background plus a glow overlay plus an engraved title plate simultaneously. The stacking preview in Section 6 shows what that combination reads like in practice.

07 Anti-patterns & Decision Criteria

Shared vocabulary for the slot picks.

When the team picks which variants to ship per slot, these are the tests each pick has to pass. The anti-patterns are also the disqualifier list — if a proposed cosmetic falls into one, redesign it before it ships.

Anti-patterns

  • Tier override. A cosmetic that changes the tier-colored chrome read (border color, notch tier tint, backdrop tier tint). Achievement-locked. Never.
  • Card louder than Pursuer. A variant where the cosmetic effect is the first thing you notice, not the Pursuer's name + level + tier. Cosmetics frame the Pursuer; they don't compete.
  • Anti-ref #4 motion. Confetti, fireworks, glitter bursts, anything that reads "mobile-game gamification." If the motion would feel at home in a casual mobile-game reward screen, it doesn't ship.
  • Inconsistent across sizes. An effect that works on the hero size but breaks the compact / mini / share / profile-header variants. Every shipped cosmetic must read correctly on all five sizes.
  • Lost-access orphan state. A cosmetic that doesn't gracefully revert when the player loses access (subscription lapses, equipped item un-purchased, seasonal unlock expires). The card should fall back to default for that slot, not break.

Decision criteria per variant

  • Reads quietly until noticed. First glance: Pursuer's identity. Second glance: the cosmetic. Never the reverse.
  • Stacks without saturating. Combining this variant with two or three other premium variants from other slots should still pass the "Pursuer first" test.
  • Tier-tints correctly. The variant's tier-color hooks (glows, gradients, accents) inherit from the tier the player earned — Bronze cosmetic ≠ Gold cosmetic in palette.
  • Holds across all five sizes. Hero / compact / profile-header / share / mini all need to render it correctly (or drop it cleanly when too small).
  • Worth what it costs. Free options should feel like meaningful upgrades over default. Premium options should feel like meaningful upgrades over free. If a tier is too close to the one below, it's wrong.
