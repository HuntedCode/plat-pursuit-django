Tally Design Workshop

The treatment for numbers, levels, XP, and milestones.

Tally turns a number from information into satisfaction. It's the headline number on the Logbook, the level on a Job row, the +XP that pops on a sync, the milestone counter that ticks past a round number. The doc names its typeface choice as "the single highest-leverage type decision" in the kit — pick wrong and every number in the product reads off. This workshop tests the four candidates side-by-side at every scale Tally appears, then layers in the dynamic pieces: count-up animation, level-up moment, fresh-decay.

01 The Showdown

Four candidates. Two references. Headline scale.

Each card renders the Pursuer Level 47 at 120px, a milestone count 1,247 at 52px, a recent gain +250 at 22px, and the inline label XP earned at 13px. The face that holds best across all four scales wins, not just the one that looks coolest at 120px.

The two reference cards (Archivo, Press Start 2P) are calibration anchors, not candidates — they show the design space's edges: how far toward "industrial signage" we could go, and how far toward "RPG / game UI" we'd be wrong to go. Use them to feel where the production picks should sit between.

What we're feeling for: character without shouting, weight without aggression, numbers that read as earned, not reported. Anti-references to keep in mind — too dashboard (anti-ref #1), too generic mobile-game (anti-ref #4), too detached / brutalist-cool (anti-ref #5).

Inter font-family: Inter

Candidate · safe baseline

47

1,247 +250 XP earned

What it is: The default sans-serif of the modern indie web. Variable, with built-in optical sizing so the same family adapts from 14px caption to 120px hero. Famously good numerals.
Risk: Anti-ref #1. Inter renders cleanly but it doesn't carry CHARACTER — a number rendered in Inter could live in any dashboard from Vercel to Linear to Stripe. The team would have to add character through animation + motion alone.

{# Card B: Space Grotesk — candidate, distinctive grotesque #}

Space Grotesk font-family: Space Grotesk

Candidate · distinctive grotesque

47

1,247 +250 XP earned

What it is: Geometric grotesque with subtle indie-tech quirk — slightly squared "4", distinctive "1" with a flag, condensed feel without being condensed. The numerals carry quiet personality.
Risk: Maxes at 700 weight (no 800/900 / black), so headlines have a touch less impact than the variable competitors. The geometric character can read slightly "design-agency" if everything else on the page is clean sans.

{# Card C: Bricolage Grotesque — candidate, quietly distinctive (Fraunces replacement) #}

Bricolage Grotesque font-family: Bricolage Grotesque

Candidate · quietly distinctive

47

1,247 +250 XP earned

What it is: Variable display sans with three parametric axes (optical size, width, weight). Personality comes from how the family moves across axes rather than from wonky letterforms — quiet character, fully tunable.
Risk: The same axes that make it powerful are easy to over-design with. Picking one Bricolage configuration to live in the system means committing to specific opsz / wdth / wght values; the wrong combo can drift to generic.

{# Card D: JetBrains Mono — candidate, ledger/data mono #}

JetBrains Mono font-family: JetBrains Mono

Candidate · ledger / data mono

47

What it is: Monospace by design — every digit is the same width naturally, no tabular-nums needed. Reads as "ledger entry" or "watchmaker's micrometer scale" — leans hard into the numbered ledger half of the explorer's-office metaphor.
Risk: Can read too tech / too code-editor. The Frame's engraving already uses mono tabular figures (first pass); going full mono everywhere risks the whole product reading as a code IDE.

03 Tabular Figures

Why every digit needs the same width.

In tabular figures, every digit occupies the same horizontal box, so columns of numbers stack cleanly — the 1 takes the same width as the 0. The Tally lives in columns more often than not (Job-level rosters, milestone trackers, XP histories), so tabular is the default. The left list below renders proportional figures (default for most faces), the right list uses tabular. Watch how the right column's digits land in the same place.

3b · Glyph Stress Test

Production numbers don't always look like 1,247. They include percent signs, plus/minus prefixes, hash marks, decimals, label prefixes (Lv. 47), and abbreviated forms (1.2k). 04 Tick-up animation

Three flavors of counting up.

The doc names "counters that count up from zero — looks arcade-y, kills weight" as an explicit anti-pattern. Tally numbers should feel earned, not frantic. Three options, each playing the same transition (1,247 → 1,297 for a +50 sync gain): a slot-machine vertical roll, a flip-card per-digit reveal, and a weighted count-up that decelerates. Click Play on each to compare. Each variant tracks its own running total, so repeated clicks keep advancing by +50.

Previewing Sections 4–6 render in Inter. Use the Pick button on any candidate in Section 1 to swap.

A — Slot-machine roll

Digits scroll vertically into place. Mechanical, sturdy.

B — Flip-card

Each digit flips on its horizontal axis. Tactile, paper-feel.

C — Weighted count

Increments rapidly, decelerates into landing. Soft pulse on settle.

05 Level-up moment

The beat that a Pursuer notices.

Not every increment is a moment. Tier completion, Pursuer Level milestones, the rare first-time-this-pursuit numbers — those deserve a beat. Old number leaves, new number rises in, a Horizon-style cyan edge pulses outward, the stage glows for a hold-of-breath count, then settles. ~1.4 seconds total. Click Trigger to play.

This draws from the Frame's motion + particle vocabulary documented in visual-identity.md §3 → Kit-level vocabulary. The level-up beat inherits that language; it doesn't invent a new one.

07 Decision Criteria

Four tests. Pass all four.

Three out of four isn't enough. One failure disqualifies. Argue against the criteria, not the candidates.

1Holds at every scale

Same face works at 96px, 48px, 22px, and 13px. Fails when it looks great big and tired small. See Section 2.

2Reads "earned," not "reported"

The number should feel like a reward, not a stat. Fails when it looks at home in a Google Analytics screenshot.

3Doesn't draw attention to itself

You should notice the number, not the letterforms. Fails when a quirky 4 or stylized 1 is the first thing you see.

4Passes the anti-refs

Avoid too dashboard (#1), too mobile-game (#4), too brutalist (#5), too twee (#6). The reference cards in Section 1 show the edges.

How to use this workshop

  • Sections 1–3 are the typography choice. Strip + cards + scales + tabular + glyph stress. Use the Pick button on a candidate to preview it in Sections 4–6.
  • Section 4 is the count-up flavor (slot / flip / weighted). The doc disqualifies the frantic count-up-from-zero variant; pick one of the three.
  • Section 5 is the celebratory beat. Reserved for meaningful moments only — Pursuer Level milestones, tier completions, the rare first-time numbers.
  • Section 6 is the ambient pacing. Every meaningful number gets a brief "fresh" period, then settles into normal weight without ceremony.
  • Section 7 is the shared rubric. Argue against the criteria, not the candidates. If a candidate fails one of the four tests, it's out.
