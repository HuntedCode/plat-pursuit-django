Manual anchor to IGDB id

For each Concept below, enter the IGDB game id you believe is the correct canonical match. The system will fetch IGDB data for that id, resolve it to its canonical form (Director's Cut / Standalone Expansion entries get collapsed to their parent), find or create the target Concept at concept_id = str(canonical_id), refresh its IGDBMatch (capturing media + Tier 1 fields), and then run the same vetting the anchor_concepts migration uses on each Game: trophy-fingerprint homogeneity vs. any existing Games in the target, plus identity cross-check (title similarity, platform overlap, release-year gap).

Clean Games move into the target via Game.add_concept. Games that flag any vetting check get a ConceptJoinReview entry the same way the migration would, so you can resolve them later via the IGDB & Enrichment → Concept join reviews admin.

Leave the IGDB id blank for any row you don't want to anchor in this run.
