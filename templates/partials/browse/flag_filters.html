{% comment %} Shared community-flag filter split-controls (house style, --pp-*). Each flag is a [ + Label - ] segmented control: + -> "Show only": games WITH this flag. - -> "Hide": games WITHOUT this flag (label struck through). Single-click semantics (browse-filters.js, [data-flag-group] hook): - Click + when neither active -> Show; when Show active -> clears; when Hide active -> swaps to Show. - - is the inverse. Form wiring: two hidden checkboxes per flag (show_X, hide_X). The +/- buttons flip these; the form submits via HTMX. On conflict the backend treats `hide` as the winner (apply_game_browse_filters). The active visual is pure CSS (.pp-flag :has(input:checked)) -- the JS only toggles the checkboxes. Required context: request (to read GET params on initial render). {% endcomment %}
