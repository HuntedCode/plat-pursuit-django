{% comment %} Shared community-flag filter split-controls. Each flag is one daisyUI `join` group of three segments: [ + ] Label [ − ] │ │ └─ Click to "Hide": games WITHOUT this flag. │ └─ Inert label, struck through when Hide is active. └─ Click to "Show only": games WITH this flag. Single-click semantics: - Click + when neither active → activates Show. - Click + when Show active → clears (back to Any). - Click + when Hide active → swaps to Show. - Same for − inversely. Form wiring: Two hidden checkboxes per flag (show_X, hide_X). The +/− buttons flip these on click; the form submits via HTMX. On conflict the backend treats `hide` as the winner (browse_helpers.apply_game_browse_filters). JS: Click handling lives in static/js/browse-filters.js via the [data-flag-group] hook. Required context: request (used to read GET params on initial render). {% endcomment %}
